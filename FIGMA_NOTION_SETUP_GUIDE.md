# Aioverse Figma & Notion Setup Guide

## Overview

This document provides comprehensive instructions for setting up the Aioverse brand assets in both Figma (visual design collaboration) and Notion (documentation and knowledge base).

**Date:** December 11, 2025  
**Version:** 1.0  
**Status:** Setup Instructions Provided

## Figma Board Setup

### File: Aioverse Brand Assets - Design System
**Status:** Created and Initialized  
**Link:** https://figma.com/design/qQo9m3NqO1H4YcS83itQuz/Aioverse-Brand-Assets---Design-System  
**Updated Link:** https://www.figma.com/design/GI2yrpDMzHWpb0lrsVxGMf/Aioverse

### Page Structure

#### 1. Overview & Instructions
- Central hub with all documentation
- Welcome message, GitHub links, quick reference

#### 2. Colors & Palette
- Primary/Secondary/Accent color swatches with hex/RGB
- Gradients, contrast ratios, WCAG compliance

#### 3. Typography & Fonts
- Font families: Nohemi (primary), TokyoTrailMono (accent), Inter (system)
- Weight variations, size scale (H1-Caption), line heights

#### 4. Logos & Marks
- Primary logo, wordmark, symbol/mark
- Color, monochrome, horizontal/vertical layouts
- Min size specs, clear space guidelines, Do's & Don'ts

#### 5. Icons & Symbols
- All sizes (24px, 32px, 48px), organized by category
- Navigation, Action, Status, Social, Utility
- Grid system, stroke weight specs

#### 6. Illustrations
- Brand character illustrations, scene compositions
- Style guide: color palette, proportion, technique

#### 7. Photography
- Style guide, color grading specs, resolution requirements

#### 8. Design Tokens
- Color tokens (visual + JSON), typography, spacing
- Border radius, shadow/elevation, motion tokens

#### 9. Component Library
- Buttons (all states/sizes), forms, cards, navigation

#### 10. Brand Guidelines
- Voice/tone, logo rules, color usage, accessibility

## Notion Workspace Setup

### Workspace: Aioverse Brand Assets

#### Databases to Create

1. **Overview** - Central hub and quick reference
2. **Colors Database** - Color name, hex, RGB, CMYK, accessibility score
3. **Typography Database** - Font name, family, weight, size range, use case
4. **Logos Database** - Type, version, format, minimum size, file links
5. **Icons Database** - Category, size, SVG/PNG files, usage
6. **Illustrations Database** - Category, color palette, use case, files
7. **Photography Database** - Category, color grading, resolution, license
8. **Design Tokens Database** - Token name, type, value, CSS/JSON formats
9. **Resources Page** - GitHub, Figma, PDF guides, tutorials
10. **Team Directory** - Name, role, email, responsibility
11. **Asset Changelog** - Date, type, change description, version

## Integration Guidelines

### GitHub ↔ Figma
- Reference GitHub asset codes in Figma components
- Use AIO-* naming conventions consistently
- Link GitHub docs in Figma file descriptions
- Version Figma library alongside GitHub releases

### GitHub ↔ Notion
- Use GitHub as source of truth for documentation
- Mirror GitHub file structure in Notion
- Sync changelog between platforms

### Figma ↔ Notion
- Embed Figma previews in Notion databases
- Use consistent naming and categorization

---

## Maintenance Schedule

- **Weekly**: Check new assets, update changelogs
- **Monthly**: Review docs, audit outdated assets, update versions
- **Quarterly**: Major brand review, team training, archive deprecated assets
